# スライドショー動画作成ツール 要件定義書

## 概要

写真フォルダと音楽ファイルをGUIで選ぶだけで、自動的にスライドショー動画（MP4）を生成するPythonアプリ。

---

## 機能要件

### 入力（GUIで設定）
| 項目 | 内容 |
|---|---|
| 写真フォルダ | JPG / JPEG / PNG 形式の画像ファイルが入ったフォルダ |
| 音楽ファイル | MP3, 3GP など FFmpeg が対応する音声ファイル |
| 出力ファイル | 保存先のMP4ファイルパス |
| 1枚あたりの秒数 | 各写真の表示時間（デフォルト: 3秒） |
| 写真の枚数 | **自動計算**（ユーザーは指定不可） |

### 出力
- MP4形式の動画ファイル
- コーデック: H.264 (libx264) / 音声: AAC
- フレームレート: 24fps
- 解像度: 1920×1080

### 動作仕様
- 写真の枚数は「音楽の長さ」と「1枚あたりの秒数」から自動計算
  - 計算式: `枚数 = floor((音楽の長さ - フェード時間) / (1枚の秒数 - フェード時間)) + 1`
  - フェード時間: 0.8秒（固定）
  - 音楽ファイルまたは1枚あたりの秒数が変更されると即座に再計算・表示
- 写真はフォルダから計算された枚数をランダムに選択
- 各写真を出力解像度（1920×1080）にリサイズしてから処理
- 写真の切り替わりはクロスフェード（0.8秒）でなめらかに遷移
  - 前の写真がフェードアウトしながら次の写真がフェードインする
  - `concatenate_videoclips(padding=-0.8)` でクリップを重ねて実現
- 音楽の末尾は自動でフェードアウト（2秒）
- 処理はバックグラウンドスレッドで実行し、GUI操作をブロックしない
- 写真読み込みとエンコードの進捗を個別のプログレスバーで表示
- 処理中に「中断」ボタンで安全にキャンセル可能
- 完了時にダイアログで通知

## GUIの構成

ウィンドウサイズ: **1100×560**（横長2カラムレイアウト）

### 左カラム（設定）
| 要素 | 説明 |
|---|---|
| 写真フォルダ入力欄 + 参照ボタン | フォルダ選択ダイアログ |
| 音楽ファイル入力欄 + 参照ボタン | ファイル選択ダイアログ |
| 出力ファイル入力欄 + 参照ボタン | 保存先選択ダイアログ |
| 1枚あたりの秒数（入力欄） | 任意の秒数を入力 |
| 写真の枚数（自動計算・読み取り専用） | 音楽と秒数から自動算出、計算式も表示 |
| 動画を作成ボタン | 処理開始（処理中は無効化） |
| 中断ボタン（赤） | 処理中にいつでもキャンセル可能 |

### 右カラム（進捗）
| 要素 | 説明 |
|---|---|
| 写真読み込みプログレスバー（青） | 何枚目かをパーセントで表示 |
| エンコードプログレスバー（赤） | フレーム単位の進捗をパーセントで表示 |
| 進捗ログエリア | 処理状況をリアルタイム表示 |

## 既知の問題と対処

### 写真の重複読み込み（対処済み）
- **原因**: `*.jpg` と `*.JPG` の両パターンで同じファイルを2重に取得していた
- **対処**: 拡張子を小文字に統一して比較し、重複を排除

### メモリ不足エラー（対処済み）
- **原因**: 高解像度写真（例: 4000×6000px = 24MP）に対してフェード処理を float64 で行うと1枚あたり約183MBのメモリを消費
- **対処**: 各写真を処理前に出力解像度（1920×1080）にリサイズすることでメモリ使用量を大幅削減

### 一時ファイルの権限エラー（対処済み）
- **原因**: MoviePy がエンコード時に作成する一時音声ファイル（`*TEMP_MPY_wvf_snd.mp4`）を相対パスで書き込もうとし、カレントディレクトリへの権限がない場合に Permission denied が発生
- **対処**: `write_videofile()` の `temp_audiofile` パラメータに出力フォルダの絶対パスを明示指定（`TEMP_slideshow_audio.mp4`）。また中断・エラー時にも一時ファイルを自動削除
